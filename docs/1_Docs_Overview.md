# 🚀 기술 문서 작성 현황 및 마일스톤

> 맞춤형 건강 관리 서비스 개발을 위한 전체 기술 문서 로드맵입니다. 파트별 병렬 개발 및 최종 캡스톤 평가를 대비하여 단계별로 문서를 관리합니다.

## 🥇 1. 기초 설계 및 필수 연동 (Phase 1)
본격적인 6인 팀(프론트/백엔드/AI)의 병렬 개발을 위해 가장 먼저 확정되어야 하는 뼈대 문서들입니다.

| 상태 | 문서명 | 상세 내용 및 목적 | 비고 & Next Step |
| :---: | :--- | :--- | :--- |
| ✅ 완료 | **🏗️ 시스템 아키텍처** | 클라이언트, 프론트/백엔드, AI, DB 전체 구조 조감도 | - |
| ✅ 완료 | **🔄 시퀀스 다이어그램** | 주요 기능(로그인, 성향 분석 등)의 시간 순서별 흐름 | - |
| ✅ 완료 | **📝 데이터 포맷** | 데이터 교환 시 사용되는 JSON 등의 통신 규격 | - |
| 🚨 시급 | **🎯 API 명세서** | 엔드포인트, Request/Response 예시, 에러 코드 | 각 파트별 독립적인 개발을 위해 최우선 작성 |
| 🚨 시급 | **🗄️ DB 스키마 & ERD** | 유저 정보, 운동/식단 데이터 간의 테이블 관계 모델링 | 정의된 포맷 기반으로 DB 테이블 구조화 |

## 🥈 2. 핵심 로직 및 기술적 깊이 (Phase 2)
단순 구현을 넘어 시스템의 '지능'과 개발자의 고민을 보여주는 문서입니다. 캡스톤 최종 평가 시 가장 깊게 질문받는 영역입니다.

| 상태 | 문서명 | 상세 내용 및 목적 | 비고 & Next Step |
| :---: | :--- | :--- | :--- |
| ✅ 완료 | **🧠 AI 아키텍처** | 모델 학습/추론 및 피드백 반영(Retraining) 파이프라인 | 필요시 상세화 |
| 🏃 진행 | **⚙️ 알고리즘 명세** | MBTI/DISC 성향 기반 맞춤 추천의 수학적/논리적 기준 | 수식 및 플로우차트 추가 권장 |
| 🏃 진행 | **⚖️ 기술 의사결정(ADR)** | Node.js, FastAPI 분리 등 기술 선택 이유와 Trade-off | 평가 및 포트폴리오 방어용 핵심 무기 |

| 상태 | 문서명 | 상세 내용 및 목적 | 비고 & Next Step |
| :---: | :--- | :--- | :--- |
| ⏳ 대기 | **☁️ 클라우드 구성도** | Oracle Cloud 내 서버 인스턴스, DB 배치 시각화 | 배포 단계 인프라 증명용 |
| ⏳ 대기 | **🛡️ 예외 처리 시나리오** | 트래픽 폭주, AI 인식 실패 시 대처 방안 등 | 서비스 완성도 및 운영 안정성 지표 |
