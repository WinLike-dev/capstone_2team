**프롬프트 구성 순서 (CHAT-12 per project decision):**
워커 AI 인풋 우선순위: 사용자 메시지 > 사용자 지시사항 > 시스템 지시사항.
- `user_message`는 Gemini `contents` 파라미터로 전달 (시스템 프롬프트 외부)
- `user_instruction`이 비어있지 않으면 시스템 프롬프트 상단에 "사용자 지시사항:" 섹션으로 포함
- 그 다음 모드별 시스템 지시사항 (아래 WORKER_BASE_PROMPT 참조)
- 마지막에 context_text (벡터 DB 검색 결과)

**WORKER_BASE_PROMPT:** docs/worker_system_instruction.txt 내용을 Python 문자열 상수로 옮긴다. 단, 모드별 출력 형식만 해당 모드의 것을 포함하도록 `_MODE_INSTRUCTIONS: dict[int, str]` 딕셔너리로 분리한다.

각 모드의 지시사항은 worker_system_instruction.txt의 해당 모드 섹션 내용 그대로 사용:
- 모드 1: 단순대화 — answer JSON
- 모드 2: 플랜 작성 — exercise plan JSON array
- 모드 3: 플랜 수정 — modified exercise plan JSON array
- 모드 4: 식단 작성 — meal plan JSON array
- 모드 5: 식단 수정 — modified meal plan JSON array
- 모드 6: 사용자 DB 수정 — updated_fields JSON
- 모드 7: 식단 기록 — calories/nutrients JSON (v1.0 /process-meal 과 동일)
- 모드 8: 추천 — exercises 3 + meals 2 JSON (v1.0 /recommend 과 동일)

**UserProfile 포매팅:** `_format_user_profile(profile: UserProfile) -> str` 헬퍼를 만들어 None이 아닌 필드만 "키: 값" 형태로 포매팅한다.

**최종 프롬프트 조립:**
```
[사용자 지시사항: {user_instruction}]  ← user_instruction이 있을 때만

## 시스템 지시사항
{공통 규칙 (우선순위, JSON 출력 등)}

## 모드 {mode} 지시사항
{_MODE_INSTRUCTIONS[mode]}

## 사용자 프로필
{formatted_profile}

## 이전 대화 맥락
{context_text}
```
